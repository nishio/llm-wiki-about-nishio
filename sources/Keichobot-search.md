# Keicho / Keichobot サーベイ

Keicho と Keichobot は同一系譜：

- 旧名: **Keicho**（2019〜）
- 新名: **Keichobot**（"KeichoからKeichobotにリネーム.md" あり）

両方を含めると約 70+ ページ（重複除く）。

## 位置付け（最重要）

[Keichobotは言語化しKozanebaは一次元化する.md](../raw/external_brain_in_markdown/pages/Keichobotは言語化しKozanebaは一次元化する.md) に明示：

- **Keichobot** = **言語化**（モヤモヤ→シンボル→関係を質問で引き出す）
- **Kozaneba** = 一次元化（関係を客体化→ストーリーへ）

→ 「**頭の中のゴチャゴチャを質問で引き出す**」ことが Keichobot の役割。
ジェンドリン哲学の「フェルトセンス」（言語化されていない気持ち）を言語化することを支援する装置。

公式コピー: 「**かんがえをひきだすチャットボットKeichobot**」「**あなたの話を聞いてくれるチャットボットKeichobot**」

## 観察

### LLM以前から「会話で言語化を促す」という設計

- 2019年起動、ルールベースで「質問→回答→次の質問」を駆動
- ChatGPT登場(2022-11)後、GPTバックエンドへ切替（2023-03）
- → **「LLMの先取り実装」** とも言える。コンセプト自体は変わらず、エンジン交代

### 「言語化」を独立フェーズとして分離した設計

KJ法系譜の中で：
- **物理KJ法**: 言語化と空間配置が混在
- **Scrapbox**: ページ作成（≒言語化）とリンク（≒関係化）が混在
- **Keichobot + Kozaneba**: 言語化（Keichobot）と一次元化（Kozaneba）を**意図的に分離**

→ プロセスの分節化が研究の核。
