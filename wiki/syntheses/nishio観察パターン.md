# nishio観察パターン

ソース横断で繰り返し現れるパターン群。**個別ページからは見えづらいが、複数のソースを突き合わせると明確になる構造**を記録する。年/10年スケール俯瞰の素材として使う。

## P1. 固定軸 + 射程拡大パターン

**「知的生産性の向上（手段）と文明の進歩（目的）」というフレーズが2018年以降のbioで定型化され、現在まで保持されている**。一方、その軸が指す対象（何の知的生産か）は段階的に拡大している。

| 時期 | 対象 |
|---|---|
| 〜2013 | プログラミング言語の進化 |
| 2014〜2018 | 機械学習・自然言語処理 |
| 2018〜2023 | 個人の知的生産技術 |
| 2023〜2024 | AI×知的生産 |
| 2024〜 | Plurality / ブロードリスニング = **集団の他者理解** |

→ 関心領域の "ピボット" ではなく、**同じレトリックで指せる範囲を広げ続ける** という戦略が見える。
意思決定として「次は何をやるか」を新規に立ち上げているわけではなく、既存軸の延長として正当化される領域に踏み込んでいる。

詳細: [[../topics/知的生産.md|知的生産]], [[../../sources/自己紹介-bios.md|自己紹介-bios]]。

## P2. ブリッジ言語パターン

新規概念を採用するとき、**既存の軸の言葉で言い換えて取り込む**。

代表例:
- ブロードリスニング ＝「**人間の他者理解能力を増強する『知的生産性向上』**」 — 2023-04-22 メモ
- Plurality ＝「**この技術が地球文明の進歩につながる鍵**」 — bio定型句
- AI協働 ＝「**人間とAIの共同作業による知的生産性向上**」 — `🌀知的生産.md` の繰り返し主題

→ 新規領域を自分の中で位置づけるための **既存軸経由の翻訳作業** が常に行われている。
これは P1（射程拡大）を成立させる言語的メカニズム。

## P3. 「これは何？」の主語ずれ

外部脳の入り口ページ `これは何？.md`（2024-09-08更新）は、**「これ＝外部脳 が何か」を説明している**。「nishioが何者か」は別ページの `自己紹介.md` に分かれ、しかも本人が「長いbioはメンテしてない」と認めている。

→ 「外部脳とは何か」と「nishioとは何者か」が**同じ場所に統合されたエントリーが存在しない**。
このWikiの `wiki/syntheses/これは何？.md` がその統合を担う設計。

## P4. NEXT本並走 = 停滞の典型

LLM以降の知的生産術を扱う「次の本」候補が複数並走している：

- LENCHI（第一章 / 第二章）
- LLMを使いこなすエンジニアの知的生産術（本体・講演資料・o1 Pro読ませる）
- NEXTエンジニアの知的生産術 / エンジニアの知的生産術NEXT
- 超・知的生産術
- エンジニアの知的生産術 増補版に向けての改善 / 加筆案

→ 単一の確定形になっていない＝**意思決定のハンドルを握ったまま意思決定していない**状態。
[[../../../../.claude/projects/-Users-nishio-claude-code-home/memory/concept_aimaina_teitai|曖昧な停滞]] の典型事例。

## P5. ファイル名規約による停滞TODOの可視化

外部脳には `TODO:` プレフィックスでファイル名が始まるページがある（例: `TODO:知的生産や学び方についての過去のアウトプットを全部まとめる.md`）。

タイトル自体が「ネクストアクションが決まると良い」のように停滞を示すページもある（例: `エンジニアの知的生産術英訳プロジェクトのネクストアクションが決まると良い.md`）。

→ **停滞TODOがファイル名で自然に表面化している** ので、ファイル名 grep だけで停滞案件をある程度抽出できる。

## P6. AI自動生成レイヤーの混在

`🌀` プレフィックスや `🤖🔁` プレフィックスのページは `omni.icon` 等によるAI生成のダイジェスト・サーベイ。
例: `🌀知的生産.md` は 2023-08〜10 にかけて日次で生成された痕跡を含む。

→ 一次ソース内に**本人記述とAI生成記述が混在**している。引用や年/10年俯瞰を作るときは区別が必要。
（このWiki自体もAI生成派生レイヤーであり、入れ子構造になっている）

## P7. 外部公開の経歴・自己紹介がメンテされていない

bio中で本人が明示的に認めている：

- 「経歴(長いもの) http://nhiro.org/ja.html」 → 「メンテしてないな〜」
- Amazon著者セントラルの古いbio → 「古いな」

→ 外部公開チャネルの自己情報は古いまま放置されている。
このWikiが**メンテ可能な代替/補助の自己紹介レイヤー**として機能する余地がある。

## このパターンリストの使い方

- 新しいキーワードを ingest するとき、これらのパターンに該当するかを確認する
- 「停滞」「ピボット」「ブリッジ」のラベルでアノテーションを残せる
- 年/10年俯瞰を作るときの目次の素材
